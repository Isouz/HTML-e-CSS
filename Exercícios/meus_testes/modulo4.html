Aula 2 – Configurando iframes.

Alguns navegadores de celular não possuem suporte para iframe. Então, tudo que colocarmos entre as tags vai aparecer para o usuario apenas se o dispositivo não tiver suporte para iframe. Nesse caso, podemos colocar uma mensagem de erro:

<iframe src="https://www.cursoemvideo.com" frameborder="0"> [Seu navegador não é compativel] </iframe>

Ou melhor, podemos simplesmente colocar o link.

<iframe src="https://www.cursoemvideo.com" frameborder="0"> <a href="https://www.cursoemvideo.com">CursoemVideo</a> </iframe>

Tamanhos.

Também podemos alterar o tamanho do iframe. Por padrão ele possui 300 x 150 (px), mas podemos adicionar um paramentro height e width dentro da tag:

<iframe src="https://www.cursoemvideo.com" frameborder="0" height="500" widht="500"> </iframe>

Também podemos redimensiona-las com o uso do CSS. Caso exista a configuração na CSS e na tag, sempre prevalecerá a CSS.

Parametro scrolling.

Esse árametro serve para habilitar ou desabilitar a barra de rolagem no elemento iframe. O atributo possui os seguintes valores:
  • auto: é o valor padrão, se o conteúdo exceder o tamanho do iframe ele cria uma barra de relagem.
  • yes: serve para exibir a barra de rolagem;
  • no: serve para esconder a barra de rolagem;
  • auto: a barra de rolagem é exibida conforme a necessidade do elemento.
<iframe src="https://www.cursoemvideo.com" frameborder="0" scrolling=""> </iframe>

frameborder.

Esse atributo é utilizado para modificar a borda padrão do elemento. Ele só aceita duas possibilidades como valor: 0 e 1. 0 ele tira a borda e 1 ele coloca a bordade 1px em volta do iframe (podemos personaliza-la na CSS usando o border).

Aula 6 – Inconvenientes do iframe.

O principal incoveniente dos iframes é que eles podem não ser acessíveis nem indexáveis por motores de busca como o Google por exemplo.

Se um motor de busca começa a "ler" sua página e chega a um iFrame, ele pula esse conteúdo. Ele não consegue entrar na URL e indexar aquela outra página, ele vai indexar separadamente. Então se uma pessoa busca o termo no site de pesquisa, pode cair direto naquele frame e não na página inteira.

Também vale ressaltar que se um leitor de tela passa lendo o conteúdo para o cego, ele também não vai conseguir ler o conteúdo de dentro do iFrame.

Existem também algumas incompatibilidades por parte dos navegadores. Por exemplo, na hora de voltar na pagina anterior, o navegador pode ficar confuso se é para voltar com a pagina principal ou com o frame.

Pode haver a possibilidade de ataque hackers pois os scripts do site usado pode ser executado dentro de nosso site. Para evitar isso vamos aprender como tornar-los mais seguros na próxima aula.

Por isso, o ideal é não abusar do uso dos iframes em nossas paginas, usando com moderação e principalmente da maneira correta, não haverá problemas.

Aula 7 – Tornando iframes mais seguros.

Para evitar problemas com ataques em nosso site através de recursos de site de terceiros, vamos aprender alguns parametros que devem ser usados dentro da nossa tag

No exemplo abaixo, temos um exemplo de um formulario de outro site.

Perceba que ao clicar em "Cadastrar", nada acontece. Isso porque usamos o parametro sandbox="sandbox" dentro da nossa tag.

Usar o parametro sandbox="sandbox" vai bloquear as ações do site dentro do iframe. Isso torna seu uso mais seguro.

O sandbox também possui outros valores:

  • sandbox="allow-same-origin": vai permitir que o pagina execute as ações desde que sejam da mesma origem de nosso site
  • sandbox="allow-forms": permitir apenas formularios.
  • sandbox="allow-scripts": permitir scripts. Como os formularios possem scripts, esse valor deve ser usado juntamente com o "allow-forms" para exibir formularios

Caso o conteúdo de do iframe precise esecutar algum script podemos usar sandbox="allow-scripts como vimos acima. Mas é muito importante não usarmos qulquer conteúdo em nosso site. No exemplo abaixo vemos um exemplo simples do uso desse parametro.

<iframe src="site.hyml" sandbox="allow-scripts">
</iframe>

Outro parametro interessante é o referrerpolicy="no-referrer". Os sites em geral usam nossos dados de navegação para exibir anúncios. O uso dessa parametro impede que o site usado colete dados de navegação do nosso usuário.

<iframe src="site.hyml" sandbox="allow-scripts" referrerpolicy="no-referrer">
</iframe>

O uso do sandbox e do referrerpolicy nos dará o maximo de segurança com o uso de outros sites dentro de nossas paginas

Aula 1 – Como criar formulários com HTML5

Formulários HTML são um dos principais pontos de interação entre um usuário e um web site ou aplicativo. Eles permitem que os usuários enviem dados para o web site.

Um formulário HTML é feito de um ou mais widgets. Esses widgets podem ser campos de texto (linha única ou de várias linhas), caixas de seleção, botões, etc.

Todo formulario deve estar entre as tag <form> e </form>. Para começar vamos fazer um exemplo de um site simples que pede nome e sobrenome do usuário. Ao lado de "Nome:" e "Sobrenome:" vamos colocar uma caixa que vai receber os dados usando a tag <input> (input:text) e nela vamos usar os parametros type, name e id. Abaixo deles vamos usar um input:submit com o value="Enviar" para criar um botão.

Veja o exemplo abaixo:

Nome:

Sobrenome:

Abaixo temos o código usado para criar este formulário:

            <form>
                <p>
                    Nome: <input type="text" name="nome" id="nome">
                </p>
                </p>
                Sobrenome: <input type="text" name="sobrenome" id="sobrenome">
                </p>
                <p><input type="submit" value="Enviar"></p>
            </form>

A tag form ainda aceita alguns parametros que melhoram sua experiência. Abaixo temos alguns deles

action

Esse parametro define a ação que será executada quando o usuário clicar no input submit. como valor vamos colocar o nome de algum arquivo de alguma linguagem de programação como PHP, JavaScript etc.

<form action="cadastro.php">

autocomplete

o parametro autocomplete serve para ligar ou desligar a sugestão de autopreenchimento que vem no input quando o usuário já utilizou essa campo anteriormente. Ele aceita dois valores, o on e o off

<form autocomplete="off">

method

É o método que o navegador usa para enviar o formulário. Os valores possíveis são:

  • post
  • get

Veremos mais sobre esse metodo nas proximas aulas.

Aula 2 – Usar Label vai melhorar seus formulários

A tag <label> é um elemento HTML que representa uma legenda para melhorar a acessibilidade de um item de interface do usuário. Este elemento também é conhecido como controle ou rótulo e pode ser associado a outro elemento de controle através de um atributo. Basicamente eles servem para "ligar" a descrição ao objeto. No nosso exemplo anterior, podemos dizer que o sobrenome é a descrição e a caixa (input) é o objeto, precisamos liga-los para que o navegador saiba que estão associados. Para fazer essa ligação, basta colocar a descrição dentro da tag <label> e no parametro form vamos colocar o nome ou id do input que faz referencia a ele:

<label for="sobrenome">Sobrenome: </label>
<input type="text" id="sobrenome">

Sempre use label nos formulários, ela é importante pois especifica qual o "rótulo" do input e ainda ajuda na experiência do usuário durante a utilização e preenchimento do formulário.

Aula 3 – Métodos GET e POST para formulários

Como vimos anteriormente, a o parametro method é método que o navegador usa para enviar o formulário. Para isso são aceitos dois valores, o get e o post.

Quando você utiliza GET, os dados são enviados na URL, existe um limite de 2048 caracteres para a requisição. Quando você utiliza o POST, os dados não ficam expostos na URL, é desta forma que se faz o envio de dados de formulário ou arquivos. Mas existe uma observação, o post não resolve os problemas de segurança, apesar dos dados não serem exibidos na URL com o seu uso, eles ainda são visiveis.

Por exemplo, se acessarmos o devtools, na aba Network, clicarmos no nome do arquivo em Payload podemos ver que os dados enviados estão descritos lá

Para proteger 100% esses dados devemos usar o protocolo HTTPS, mas esse já é um assunto bem mais avançado.

<form method="GET">
<form method="POST">
