Aula 2 – Configurando iframes.

Alguns navegadores de celular não possuem suporte para iframe. Então, tudo que colocarmos entre as tags vai aparecer para o usuario apenas se o dispositivo não tiver suporte para iframe. Nesse caso, podemos colocar uma mensagem de erro:

Ou melhor, podemos simplesmente colocar o link.

<iframe src="https://www.cursoemvideo.com" frameborder="0"> <a href="https://www.cursoemvideo.com">CursoemVideo</a> </iframe>

Tamanhos.

Também podemos alterar o tamanho do iframe. Por padrão ele possui 300 x 150 (px), mas podemos adicionar um paramentro height e width dentro da tag:

<iframe src="https://www.cursoemvideo.com" frameborder="0" height="500" widht="500"> </iframe>

Também podemos redimensiona-las com o uso do CSS. Caso exista a configuração na CSS e na tag, sempre prevalecerá a CSS.

Parametro scrolling.

Esse árametro serve para habilitar ou desabilitar a barra de rolagem no elemento iframe. O atributo possui os seguintes valores:
  • auto: é o valor padrão, se o conteúdo exceder o tamanho do iframe ele cria uma barra de relagem.
  • yes: serve para exibir a barra de rolagem;
  • no: serve para esconder a barra de rolagem;
  • auto: a barra de rolagem é exibida conforme a necessidade do elemento.
<iframe src="https://www.cursoemvideo.com" frameborder="0" scrolling=""> </iframe>

frameborder.

Esse atributo é utilizado para modificar a borda padrão do elemento. Ele só aceita duas possibilidades como valor: 0 e 1. 0 ele tira a borda e 1 ele coloca a bordade 1px em volta do iframe (podemos personaliza-la na CSS usando o border).

Aula 6 – Inconvenientes do iframe.

O principal incoveniente dos iframes é que eles podem não ser acessíveis nem indexáveis por motores de busca como o Google por exemplo.

Se um motor de busca começa a "ler" sua página e chega a um iFrame, ele pula esse conteúdo. Ele não consegue entrar na URL e indexar aquela outra página, ele vai indexar separadamente. Então se uma pessoa busca o termo no site de pesquisa, pode cair direto naquele frame e não na página inteira.

Também vale ressaltar que se um leitor de tela passa lendo o conteúdo para o cego, ele também não vai conseguir ler o conteúdo de dentro do iFrame.

Existem também algumas incompatibilidades por parte dos navegadores. Por exemplo, na hora de voltar na pagina anterior, o navegador pode ficar confuso se é para voltar com a pagina principal ou com o frame.

Pode haver a possibilidade de ataque hackers pois os scripts do site usado pode ser executado dentro de nosso site. Para evitar isso vamos aprender como tornar-los mais seguros na próxima aula.

Por isso, o ideal é não abusar do uso dos iframes em nossas paginas, usando com moderação e principalmente da maneira correta, não haverá problemas.

Aula 7 – Tornando iframes mais seguros.

Para evitar problemas com ataques em nosso site através de recursos de site de terceiros, vamos aprender alguns parametros que devem ser usados dentro da nossa tag

No exemplo abaixo, temos um exemplo de um formulario de outro site.

Perceba que ao clicar em "Cadastrar", nada acontece. Isso porque usamos o parametro sandbox="sandbox" dentro da nossa tag.

Usar o parametro sandbox="sandbox" vai bloquear as ações do site dentro do iframe. Isso torna seu uso mais seguro.

O sandbox também possui outros valores:

  • sandbox="allow-same-origin": vai permitir que o pagina execute as ações desde que sejam da mesma origem de nosso site
  • sandbox="allow-forms": permitir apenas formularios.
  • sandbox="allow-scripts": permitir scripts. Como os formularios possem scripts, esse valor deve ser usado juntamente com o "allow-forms" para exibir formularios

Caso o conteúdo de do iframe precise esecutar algum script podemos usar sandbox="allow-scripts como vimos acima. Mas é muito importante não usarmos qulquer conteúdo em nosso site. No exemplo abaixo vemos um exemplo simples do uso desse parametro.

<iframe src="site.html" sandbox="allow-scripts">
</iframe>

Outro parametro interessante é o referrerpolicy="no-referrer". Os sites em geral usam nossos dados de navegação para exibir anúncios. O uso dessa parametro impede que o site usado colete dados de navegação do nosso usuário.

<iframe src="site.html" sandbox="allow-scripts" referrerpolicy="no-referrer">
</iframe>

O uso do sandbox e do referrerpolicy nos dará o maximo de segurança com o uso de outros sites dentro de nossas paginas

Aula 1 – Como criar formulários com HTML5

Formulários HTML são um dos principais pontos de interação entre um usuário e um web site ou aplicativo. Eles permitem que os usuários enviem dados para o web site.

Um formulário HTML é feito de um ou mais widgets. Esses widgets podem ser campos de texto (linha única ou de várias linhas), caixas de seleção, botões, etc.

Todo formulario deve estar entre as tag <form> e </form>. Para começar vamos fazer um exemplo de um site simples que pede nome e sobrenome do usuário. Ao lado de "Nome:" e "Sobrenome:" vamos colocar uma caixa que vai receber os dados usando a tag <input> (input:text) e nela vamos usar os parametros type, name e id. Abaixo deles vamos usar um input:submit com o value="Enviar" para criar um botão.

Veja o exemplo abaixo:

Nome:

Sobrenome:

Abaixo temos o código usado para criar este formulário:

            <form>
                <div>
                    Nome: <input type="text" name="nome" id="nome">
                </div>
                </div>
                    Sobrenome: <input type="text" name="sobrenome" id="sobrenome">
                </div>
                <div>
                    <input type="submit" value="Enviar">
                </div>
            </form>

A tag form ainda aceita alguns parametros que melhoram sua experiência. Abaixo temos alguns deles

action

Esse parametro define a ação que será executada quando o usuário clicar no input submit. como valor vamos colocar o nome de algum arquivo de alguma linguagem de programação como PHP, JavaScript etc.

<form action="cadastro.php">

autocomplete

o parametro autocomplete serve para ligar ou desligar a sugestão de autopreenchimento que vem no input quando o usuário já utilizou essa campo anteriormente. Ele aceita dois valores, o on e o off

<form autocomplete="off">

Esse parametro também pode ser usadno dentro do input, nesse caso, ele vai exibir uma serie de outras opções como valor além de "on" e "off", como senha, usuário, cep etc. Isso ajuda o navegador do usuário que utuliza gerenciadores de senhas por exemplo, assim o navegador vai sugerir o autopreenchimento de uma forma mais precisa.

method

É o método que o navegador usa para enviar o formulário. Os valores possíveis são:

  • post
  • get

Veremos mais sobre esse metodo nas proximas aulas.

Aula 2 – Usar Label vai melhorar seus formulários

A tag <label> é um elemento HTML que representa uma legenda para melhorar a acessibilidade de um item de interface do usuário. Este elemento também é conhecido como controle ou rótulo e pode ser associado a outro elemento de controle através de um atributo. Basicamente eles servem para "ligar" a descrição ao objeto. No nosso exemplo anterior, podemos dizer que o sobrenome é a descrição e a caixa (input) é o objeto, precisamos liga-los para que o navegador saiba que estão associados. Para fazer essa ligação, basta colocar a descrição dentro da tag <label> e no parametro for vamos colocar o nome ou id do input que faz referência a ele:

<label for="sobrenome">Sobrenome: </label>
<input type="text" id="sobrenome">

Sempre use label nos formulários, ela é importante pois especifica qual o "rótulo" do input e ainda ajuda na experiência do usuário durante a utilização e preenchimento do formulário.

Aula 3 – Métodos GET e POST para formulários

Como vimos anteriormente, a o parametro method é método que o navegador usa para enviar o formulário. Para isso são aceitos dois valores, o get e o post.

Quando você utiliza GET, os dados são enviados na URL, existe um limite de 2048 caracteres para a requisição. Quando você utiliza o POST, os dados não ficam expostos na URL, é desta forma que se faz o envio de dados de formulário ou arquivos. Mas existe uma observação, o post não resolve os problemas de segurança, apesar dos dados não serem exibidos na URL com o seu uso, eles ainda são visiveis.

Por exemplo, se acessarmos o devtools, na aba Network, clicarmos no nome do arquivo em Payload podemos ver que os dados enviados estão descritos lá

Para proteger 100% esses dados devemos usar o protocolo HTTPS, mas esse já é um assunto bem mais avançado.

<form method="GET">
<form method="POST">

Aula 04 – Criando caixas de texto e de senha

Como vimos anteriormente, existem varios tipos de inputs, se você estiver ceiando um formulário que vai receber a senha do seu visitante podemos utilizar o input:password. O argumento password faz com que a senha digitada não seja exibida na tela, ao invés de exibir os caracteres o navegador exibe bolinhas pretas, alguns navegadores exibem asteríscos (*):

<input type="password" id="senha">

Outro botão que pode ser muito útil é o input:reset que basicamente limpa os campos quando o usuário clica.

Se tratando de login com usuário e senha, sabemos que inserir os dados é obrigatório para que o usuário possa acessar sua conta. Para sinalizar ao usuário que determinado compo é obrigatório para prosseguir, devemos usar o parametro required, ele vai exibir um "balão" automaticamente quando o usuário tentar prosseguir sem preencher os dados solicitados:

<input type="text" id="usuario" required>
<input type="password" id="senha" required>

Comprimento

Podemos também colocar quantidade minima e maxima dos compimentos do campos, para isso basta usar os parametros minlenth e maxlenth seguidos da quantidade de caracteres

<input type="text" id="usuario" maxlenth="15">
<input type="password" id="senha" minlenth="8">

Tamanho da caixa.

Você também pode mudar o tamanho da caixa usando o parametro size e no valor é só definir o tamanho baseado em quantos caracteres serão exibidos

<input type="text" id="usuario" size="10">

placeholder

O placeholder é usado para inserir um texto dentro da caixa com um pouco de transparenica. Vale ressaltar que ele não preenche dados no campo, funciona mais como uma "dica" para o usuário.

<input type="text" id="usuario" placeholder="11">

Aula 05 – Elementos number, month, date e time.

Como podemos perceber, existem varios elementos para formulários em HTML, mas nesse curso não será possivel ver todos, devido ao grande numero de elementos e possibilidades, mas separamos aqui alguns mais udados além daqueles que já vimos nas aulas anteriores:

  • number: (input:number) Só aceita números ou pontos e traçõs, letras não são aceitas. Esse elemento possuem dois elementos importantes que podem ser úteis, o min, o max e o step que definem o número minimo, maximo que poderá ser digitado e o passo ou "salto" entre as numerações.
    <input type="number" id="media" min="0" max="10" step="0.5">
  • month: (input:month) Cria uma caixa especial que permite que o usuario selecione ou digite o mês e o ano.
    <input type="month">
  • date: (input:date) Cria uma caixa que permite que o usuário digite apenas o dia/mês/ano.
    <input type="date">
  • time: (input:time) Cria uma caixa que permite que o usuário digite apenas o o horário.

Em todos eles podemos usar o parametro value que vai preencher alguma informação por padrão. Por exemplo, se você precisar que a média do campo já venha com o número 5, podemos usar o value=5 dentro da tag, dessa forma o placeholder não será exibido.

<input type="number" id="media" value="5">

Vale ressaltar também que os navegadores podem funcionar de maneiras diferentes, as coisas podem mudar um pouco ou até mesmo não funcionar da mesma maneira. O importante é testar no maior número de namegadores e dispossitivos possiveis para garantir que se site não apresente problemas que possam te prejudicar.

Aula 08 – Checkbox e Radio button em HTML

Checkboxes e radio buttons são elementos que possibilitam ao usuário escolher uma ou várias opções oferecidas pelo formulário. Pode-se selecionar vários checkboxes ao mesmo tempo, mas apenas um radio button (quando um é selecionado, os outros são desmarcados).

Checkbox

Para usa-lo vamos inserir a tag input com o tipo checkbox (input:checkbox). Podemos inserir quantas opções forem necessárias.

Radio.

Para usa-lo vamos inserir a tag input com o tipo radio (input:radio). Podemos inserir quantas opções forem necessárias.

O detalhe aqui é que o radio se diferencia do checkbox pois, como dito anteriormente, ele pertite selecionar apenas uma opção, caso tente selecionar outra caixa, a anterior será desmarcada automaticamente.

Para que isso funcione corretamente, devemos lembrar de comocar o mesmo valor no parametro name.

<input type="radio" name="sexo" id="imas" value="M">
<label for="imas" > Masculino </label><br>

Observações importantes.

  • Outra coisa que pode ser útil é o parametro checked que faz com que uma opção já venha marcada por padrão. Mas se tradando de radio podemos usar o checked em apenas uma das opções.
    <input type="checkbox" name="esfut" id="iesfut" checked>

    O elemento de saída (<output>) é um elemento no qual um site ou aplicativo pode injetar os resultados de um cálculo ou o resultado de uma ação do usuário. Para isso, vamos usar um pouquinho de JavaScript. Não vamos entrar a fundo em JavaScript aqui, vamos apenas mostrar alguns elementos basicos que nos auxiliam para exibir determinados resultados na tela.

    No exemplo abaixo temos dois imputs do tipo "number", cada um recebe um número que são somados e o resultado é exibido na saída "Soma"



    0

    Para fazer isso, usamos a tag <output> com um id para exibir o resultado e dentro dos inputs vamos inserir o parametro oninput que vai receber o seguinte código no valor: "isoma.innerHTML = Number(inum1.value) + Number(inum2.value)"

    Veja o código abaixo para entener melhor:

    <label for="inum1">Número 1:</label>

    <output id="isoma">0</output>

    O detalhe é que esse método não envia a soma, ele apenas calcula e exibe na tela.

    Aula 01 – O que são Media Queries em CSS

    Uma media query consiste de um media type e pelo menos uma expressão que limita o escopo das folhas de estilo, tal como largura, altura e cor. Media queries, adicionadas no CSS3, deixam a apresentação do conteúdo adaptado a uma gama especifica de dispositivos não precisando mudar o conteúdo em si.

    Veja as imagens abaixo:

    Perceba como o site do you tube muda de acordo com o tamanho da tela. A quantidade de videos que é exibido se adapta, assim como o formato do menu lateral e até a barra de pesquisa. Essa adaptação é possivel por conta das Media queries.

    Aula 02 – Criando um site com versão para impressora

    Existe também media query para impressão, na aula anterior vimos que um bom site pode se adaptar a qualquer tamanho de tela, isso também deve ser aplicado para impressões.

    Para aplicar a media query vamos criar um arquivo separado em CSS dentro da nossa tag <head>. Começamos como um link normal para CSS, a diferença é que no final vamos usar o parametro media com o valor desejado, nesse caso, como o nosso exemplo é para impressoras, vamos usar o valor print.

    Para as demais configurações de tela no nosso site, podemos criar um link para o arquivo CSS mas com o valor screen

    E para as configurações gerais, ou seja, as configurações comuns entre tela e impressão, podemos usar o valo all.

    <link rel="stylesheet" href="style.css" media="all">
    <link rel="stylesheet" href="style.css" media="screen">
    <link rel="stylesheet" href="style.css" media="print">

    Dessa forma, separamos e organizamos os estilos de nossa pagina em telas ou em impressões, facilitando até futuras manutenções em nossas paginas.Não vamos notar a diferença durante a negação na pagina, mas quando nessessário imprimir a pagina, as configurações exibidas serão diferentes.

    Lembrando que, como vimos no módulo 2, as serífas facilitam muito na hora de ler algo em papel e, portanto, pode ser interessante usarmos uma fonte serifada na hora de configurarmos a media query para impressão. Além disso, exibir o menu em uma impressão, pode não ser muito útil, por isso, podemos configurar nossa fora de eslilo para impressão de forma com que o menu seja retirado da exibição (display:none;)

    São muitas possibilidades, o importante é pensar em criar um conteúdo que ficará bom para futuras e manutenções e principalmente para o consumo dos usuários.

    Aula 03 – Múltiplas Media Features com CSS.

    Na aula anterior vimos a media="all", media="screen" e a media="print" todas são media types (tipos de média).

    Falando de screen por exemplo, sabemos que, dentro dessa categoría, existem vários tipos de tela como celular, tablet, pc, etc; e é aí que entram as medias features.

    Para usar as medias features escrevemos and(e) após a media type e a caracteristica (orientação da pagina) colocamos entre parenteses entre parênteses, veja o exemplo:

    <link rel="stylesheet" href="style.css" media="screen" and (orientation:portrait)>
    <link rel="stylesheet" href="style.css" media="screen" and (orientation:landscape)>

    O primeiro link (orientation:portrait) é referente ao arquivo das configurações para as telas em modo retrato, ou seja, em pé. O segundo (orientation:landscape) é para as configuraçõespara telas em modo paisagem, ou seja, deitado.

    Media query é basicamente a combinação de media type + media features (tipo de mídia + características de midia).

    Aula 06 – Mobile First

    Mobile First é um conceito aplicado em projetos web onde o foco inicial da arquitetura e desenvolvimento é direcionado aos dispositivos móveis e em seguida para os desktops.

    Em projetos web de uma forma geral o foco precisa ser no usuário para que ele tenha a melhor experiência possível e cumpra sua missão no website.

    Quando projetamos primeiramente para mobile somos forçados a sermos sucintos e fazer uso da melhor maneira dos poucos espaços disponíveis nas telas dos dispositivos móveis e auxiliar os usuários a realizarem suas desejadas tarefas.
